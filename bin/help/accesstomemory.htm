2.3. Access to memory


The memory has more than 50 numeric variables, but each and everyone of them has a specific role in the program. Some functionalities use some of this variables to store their results, ignoring if it has already a value or not. Other variables have the name of its purpose - e.g., stddev is a variable used for statistical analysis, so you must not save values in it because you'll lose your data.


You can access any of these variables via the calculator, and you can either read then or write on them (except for constants, which are read-only). For that, you must just enter the name of the variable: for example, the expression "2+a" has a reference for a rtFloat variable, so Mini Calc will replace the identifier with its value.


Whenever you introduce an expression that returns a rtFloat or a rtInteger, the variable ans is updated with its result.


Mini Calc cannot store its data on ten particular memories which are reserved for the user - mem1 to mem10. This memories are the most secure of all once Mini Calc has no write-access to them except when the user explicitly orders it to access them.



Reading from a variable


If you need to use a value from a variable, just write its identifier - Mini Calc will detect it and replace it with its value.


Some variables are, in fact, constants - they are variables specially identified to not be accessed by any means. If you try to save data in this constants, Mini Calc will just ignore it.




Writing on a variable


From the calculator, you can attribute values to variables, i.e., you can save data by writing on them. For that, there is a special command that allows you to do it:


save.identifier expression


If you want to save the result of the expression "2+tan(degtorad(45))" into the variable mem1, you must type this:


save.mem1 2+tan(degtorad(45))


Mini Calc will inform you if the data was successfully saved by showing the message «Saved in "mem1"».
