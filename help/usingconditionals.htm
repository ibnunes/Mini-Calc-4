2.3.6. Using conditionals


Conditionals are a special feature of Mini Calc which allows to return different values based on one condition. For example, if x<0 we want to calculate x/2-5, but if x>0 we rather want x/2+5.



is(condition : rtBoolean) : rtInteger


This function is essentially a converter with a very special behavior - it only returns two possible values:

  • If the condition os True, then it'll return 1;
  • Otherwise, it'll return 0.


So, you only have to multiply the part of the equation in which you want a condition by this function. For the given example, we would have this:


(x/2-5)*is(x<0) + (x/2+5)*is(x>0)


When the condition is False, the function will return 0, and as any number multiplied by zero is zero, the respective term is abolished.


However, when you're using the graphical calculator, you'll see that, between these two conditions, there is a visible "jump".




There is a little trick available in Mini Calc to make that jump disappear.


The graphical calculator computes each point to be displayed and determines if the point is valid. If not, it'll not be drawn. As the is makes the function return 0 when all conditionals fail - and 0 is a valid point -, we have to "fool the system" and force it to believe that the point is invalid so it won't be displayed.


With that purpose there is this function:



is!(condition : rtBoolean) : rtInteger


  • If the condition is False, it returns 1 - this is totally different from the is function!
  • If the condition is True, it will raise an error (Division By Zero). Because of this error, the graphical calculator assumes that the point is invalid and it isn't drawn.


Therefore, the condition is determine the points where there isn't a definition in the function.


So, lets add this "tweak" - at x=0 we have no definition, so we'll add this with the is! function:



(x/2-5)*is(x<0) + (x/2+5)*is(x>0) * is!(x=0)


And now the resulting graphic is this:




Be careful, though! Depending on the zoom definitions, the graphical calculator can jump this conditional and the "jump" gets visible. In other cases, the system can refuse to assume the equation for some unexpected error with this conditional - this is normal since we are tweaking the system forcing the generation of errors.
